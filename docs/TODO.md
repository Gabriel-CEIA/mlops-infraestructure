# Posibles mejoras y sugerencias

- [ ] **Usar una VPN de uso para la universidad:** De este modo la infraestructura se encontrará menos expuesta. Se puede contratar un servicio o también posible implementar en una instancia de EC2, aunque esta última opción requeririía de mantenimiento manual del servicio.
- [ ] **Considerar el uso de ECS en lugar de instancias EC2:** Ya que se usan imagenes docker tanto para el servidor de despliegue como para los entrenamientos y que [MLFlow provee una imagen Docker e intrucciones de despliegue en el servicio de ECS](https://mlflow.org/docs/latest/self-hosting/deploy-to-cloud/aws/), puede ser una opción interesante a considerar al eliminar la necesidad de manejar instancias EC2 manualmente. 
Pero como desventaja puede resultar en una configuración de terraform más compleja y en mayores costos a requerir del servicio de Amazon Elastic Load Balancer (ELB) para exponer los contenedores a la red.
- [ ] **Agregar autenticación e implementar workspaces en MLFlow:** De momento, cualquier persona que tenga acceso a la URI de MLFlow puede acceder a la plataforma, ver todos los experimentos registrados, hacer cambios, etc.
MLFlow provee herramientas para implementar [autenticación de usuarios](https://mlflow.org/docs/latest/self-hosting/security/basic-http-auth/), restringiendo el acceso a la plataforma, además de restringir los accesos y permisos de los usuarios. Se pueden combinar con la caracteríastica de [workspaces](https://mlflow.org/docs/latest/self-hosting/workspaces/permissions/) para tener un control más granular sobre que tienen acesso los usuarios.
- [ ] **Agregar nuevas funciones lambda:** Es recomendable definir otras funciones lambda para poder crear instancias de entrenamiento más acordes a las necesidades de cada proyecto.
- [ ] **Implementar la eliminación automática de las instancias de entrenamiento:** De momento solo se detiene su ejecución. Pero lo ideal sería eliminarlas, ya que después de su ejecución, ya no son necesarias.
- [ ] **Implementar la monitoreo con cloudwatch:** De momento, solo las intancias de entrenamiento tienen monitoreo de logs. Idealmente sería bueno monitorear las intancias de MLFlow y el servidor de despliegue para estar al tanto de problemas que se puedan tener. Además de levantar un dashboard para facilitar este monitoreo.
